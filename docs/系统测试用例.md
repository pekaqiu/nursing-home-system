## 3. 界面测试

1. 系统页面、菜单、按钮、表单和提示信息均采用中文显示。
2. 普通用户端页面字体较大，按钮明显，表单输入框高度较高，便于老年用户浏览和操作。
3. 表单提示、错误提示和成功提示使用正式中文文案，状态说明明确。
4. 页面配色对比清晰，重要入口通过卡片和按钮进行突出显示。
5. 页面布局简洁，没有复杂动画，整体适合老年用户使用，也便于论文截图展示。

## 4. 数据库测试

通过功能操作后，应重点检查以下数据是否能够正常写入数据库：

1. 用户注册数据是否写入 `sys_user` 表
2. 入住申请数据是否写入 `checkin_apply` 表
3. 审核状态和审核备注是否写回 `checkin_apply` 表
4. 老人档案数据是否写入 `elderly_profile` 表
5. 床位状态是否写回 `bed_info` 表
6. 护理记录数据是否写入 `care_record` 表

## 5. 测试结论

通过功能测试、界面测试和数据库测试可以看出，系统主要功能运行正常，能够完成普通用户入住申请、管理员审核、工作人员入住登记和护理记录管理等核心业务流程。系统页面显示正常，提示信息清晰，普通用户端在字体、按钮和页面布局方面体现了一定的适老化设计，基本满足本科毕业设计的系统实现与演示要求。
